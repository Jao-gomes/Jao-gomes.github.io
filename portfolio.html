01 — Front-end🥘🍷

App de Receitas

Este foi um dos projetos mais legais que desenvolvi até o momento, realizado em equipe, com foco no desenvolvimento do front-end de uma aplicação de delivery. O projeto consistiu na criação de uma plataforma para uma distribuidora de bebidas que buscava digitalizar e escalar suas operações. A aplicação permite que diferentes tipos de usuários acessem o sistema por meio de login, com funcionalidades específicas para clientes, pessoas vendedoras e administradoras. Durante o desenvolvimento, tive maior participação na implementação de testes, garantindo a qualidade e o correto funcionamento da aplicação. Atuei na criação de testes automatizados para validar componentes, fluxos de usuário e comportamentos da interface, contribuindo para a redução de erros e maior confiabilidade do sistema. Também seguimos boas práticas como componentização, organização de código e consumo de APIs, garantindo uma aplicação escalável e de fácil manutenção. Esse projeto foi essencial para fortalecer minhas habilidades em desenvolvimento front-end, além de aprimorar minha comunicação e colaboração em equipe durante todo o processo de desenvolvimento.

03 — Back-end⚽️

Trybe Futebol Clube

Eu desenvolvi o TFC, um site informativo sobre partidas e classificações de futebol. Atuei no desenvolvimento da API utilizando a metodologia TDD, além de integrar as aplicações por meio do Docker Compose, garantindo o funcionamento completo do sistema com consumo de um banco de dados. Nesse projeto, construí um back-end dockerizado utilizando modelagem de dados com Sequelize. Todo o desenvolvimento seguiu as regras de negócio previamente definidas, e a API foi estruturada para ser consumida por um front-end já disponibilizado. Implementei autenticação baseada em token, garantindo que apenas pessoas usuárias autenticadas pudessem adicionar ou alterar partidas. Também desenvolvi o relacionamento entre as tabelas de times e partidas, permitindo a atualização correta das informações. Além disso, apliquei regras de negócio no back-end para garantir que a tabela de classificação exibida no front-end fosse populada de forma correta e consistente para a pessoa usuária.

04 — Front-end🔑

Password Manager

Neste projeto, desenvolvi um gerenciador de senhas com foco em manipulação de estado e construção de interfaces dinâmicas no front-end. A aplicação permite que a pessoa usuária cadastre, visualize, oculte e remova senhas de diferentes serviços, garantindo uma experiência simples e funcional para gerenciamento de dados sensíveis. Durante o desenvolvimento, trabalhei com conceitos fundamentais do desenvolvimento front-end, como controle e atualização de estado, renderização condicional de componentes e criação de formulários interativos. Também implementei a listagem dinâmica de dados e a comunicação entre componentes, utilizando props e callbacks para garantir o fluxo correto de informações entre componentes pais e filhos. Esse projeto foi importante para consolidar minha base em construção de interfaces com comportamento dinâmico e organização de componentes reutilizáveis.

05 — Banco de dados🏦🎲

All For One

No projeto All For One, eu pratiquei e consolidei todos os conceitos de SQL aprendidos até então, utilizando o banco de dados Northwind. Trabalhei na resolução de uma série de desafios com diferentes níveis de complexidade, escrevendo queries SQL para atender a cada requisito proposto. Para isso, criei arquivos individuais para cada desafio, organizando o código de forma estruturada e seguindo as boas práticas exigidas. Utilizei consultas envolvendo seleção de dados, filtros, ordenação, agregações e junções entre tabelas, sempre referenciando corretamente o banco de dados e suas tabelas. Além disso, versionei todo o desenvolvimento utilizando Git e GitHub, entregando o projeto por meio de um Pull Request com todos os desafios implementados.

06 — DevOps🐋🐳

Docker to-do-list

Neste projeto, eu conteinerizei aplicações, criei a comunicação entre elas e orquestrei todo o seu funcionamento utilizando Docker. Trabalhei com uma aplicação full-stack de lista de tarefas, estruturando a conteinerização das três frentes do sistema: front-end, back-end e uma aplicação de testes responsável por validar a comunicação entre os serviços. Desenvolvi as imagens Docker para cada parte da aplicação e configurei sua execução utilizando o Docker Compose, garantindo que todos os serviços funcionassem de forma integrada. Além disso, utilizei diversos comandos do Docker com diferentes níveis de complexidade, organizando cada um deles em arquivos individuais conforme os requisitos do projeto. Estruturei esses comandos dentro do diretório específico, respeitando a padronização de nomenclatura e o contexto de execução definido. Por fim, configurei o arquivo de orquestração docker-compose.yml, garantindo que as aplicações fossem executadas corretamente em conjunto, com comunicação eficiente entre os containers.
